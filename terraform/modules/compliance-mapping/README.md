# Compliance Mapping Module

## Purpose

The `compliance-mapping` module maps SAMSON technical controls to recognized compliance and security frameworks.

It provisions no infrastructure. It provides structured Terraform outputs that connect implemented safeguards to governance, risk, and compliance requirements.

## Frameworks

- NIST 800-53
- ISO 27001
- HIPAA Security Rule
- CIS Controls

## Mapped Controls

- Terraform remote state encryption
- S3 state versioning
- Corporate tagging standards
- Budget guardrails
- SCP region restrictions
- SCP root-user restrictions
- SCP validation
- Policy simulation

## Business Value

This module translates cloud security engineering into audit-ready governance language. It helps explain not only what was built, but why each control matters from a risk and compliance perspective.

## SAMSON Role

SAMSON treats security controls as traceable governance artifacts. This module provides the compliance context needed for executive reporting, audit evidence, and architecture defense.
